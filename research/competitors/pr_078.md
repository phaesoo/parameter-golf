# PR #78: 8192 Vocab Size, NorMuon, Selective Quantization

- **Author**: mtybadger (Spruce Campbell)
- **BPB**: 1.1858 (post-quant), 1.1717 (pre-quant)
- **Date**: 2026-03-19
- **Status**: OPEN
- **Model size**: 14.1MB (14,743,224 model + 53,612 code) — 1.9MB headroom
- **Params**: not specified, but larger embedding (8192×512=4.19M) offset by 8 layers instead of 9

## Techniques

### 1. Vocab 8192 (biggest contributor, est. -0.025 to -0.030 BPB)
- 8x larger tokenizer → more bytes per token → lower tokens_per_byte → lower BPB
- Tradeoff: embedding table 524K→4.19M params, had to drop 1 layer (9→8)
- Retokenized FineWeb dataset with new SentencePiece BPE model
- seq_len increased to 4096 to match longer byte-coverage per sequence

### 2. NorMuon Optimizer (est. -0.005 to -0.010 BPB)
- Adds per-row second-moment normalization to Muon
- Maintains beta2=0.95 momentum of per-row variance
- Adaptive step size: `1 / sqrt(second_momentum)`
- Re-normalizes to preserve original update norm
- Essentially Adam-like per-dimension LR on top of Muon's spectral normalization

### 3. Selective Quantization w6e8 (enabler, -0.014 BPB degradation)
- Transformer weight matrices: 6-bit quantization
- Embedding table: 8-bit quantization (larger, more sensitive)
- Custom bitpacking: `pack_lowbit_tensor` at 6 bits/value (25% savings over int8)
- Pre-quant 1.1717 → post-quant 1.1858 = 0.014 BPB penalty

### 4. Other Changes
- TRAIN_SEQ_LEN: 1024→4096 (slower steps: 64ms vs 43ms, fewer steps: 9359)
- WARMDOWN_ITERS: 1200→3000 (adapts to fewer total steps)
- NUM_LAYERS: 9→8 (room for larger embedding)

## Novel Ideas

1. **Vocab-vs-depth tradeoff under fixed size budget**: losing 1 layer hurts, but BPB rewards bytes-per-token so heavily that net effect is positive. This is specific to BPB scoring.
2. **NorMuon's norm-preserving adaptive scaling**: second-moment normalization without disturbing Muon's spectral properties.
3. **Asymmetric quant precision (w6e8)**: embeddings deserve higher precision than transformer weights.
4. **1.9MB headroom remaining**: could fit more parameters or higher precision.

## Key Numbers

- No sliding window eval used → significant untapped improvement available
- Quant penalty 0.014 is high → STE fake-quant during training would reduce this
- 1.9MB headroom → could add back the 9th layer or widen MLP

## Takeaways

- Vocab size is a legitimate major lever for BPB scoring
- This submission does NOT use sliding window eval — combining with sliding window could push to ~1.15
- NorMuon is a drop-in Muon replacement worth testing
- The quant gap (0.014) suggests STE training would help significantly
