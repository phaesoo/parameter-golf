## Key Insights

### Mixed-Precision Quantization (most novel idea)
- Block weights (CastedLinear): int6 with STE fake-quant during training → only +0.0015 BPB penalty
- Token embedding (nn.Embedding): int8 — no STE protection during training, so needs gentler quant
- Previous uniform int6 caused +0.048 BPB penalty, dominated by embedding degradation
- int6 stored in int8 containers, zlib compresses the zero high bits

### Shorter training seq = more throughput
- seq_len 4096→1024: attention O(N^2) savings → step time 55.5ms→48.4ms
- Result: 12,395 steps (vs 10,804), 53% more training tokens
- Rationale: 512-dim model can't meaningfully exploit 4K context anyway

### Sliding window eval (stride=64)
- Each scored token sees 960 tokens of context (vs ~512 average in chunked eval)
- Eval time: ~73s (within eval budget, outside training budget)
- +0.0335 BPB improvement — biggest single technique

### Wider MLP (3x expansion)
- hidden=1536 (64-aligned for H100)
- Only possible because int6 saves enough bytes for the larger model

## Novel Ideas

1. **STE-aware quantization asymmetry**: embedding gets int8 because it lacks STE training, block weights get int6 because STE made them robust. Principled, not a hack.
2. **Wider-model-fits-because-of-better-quant feedback loop**: int6 enables 21.8M params in 16MB, which enables wider MLP, which improves BPB.
3. **Counter-intuitive seq_len reduction**: shorter training context → more tokens seen → better model despite less context per sequence.

## Takeaways

- Sliding window eval (~0.034) + wider MLP (~0.029) account for nearly all improvement
- Mixed quant is an enabler (fit bigger model) with near-zero quality cost
- Architecture is barely changed from baseline — just MLP width
