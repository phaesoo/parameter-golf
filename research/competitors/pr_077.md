## Key Insights

### Document Isolation (-0.011)
- Evaluate each document independently, reset context at BOS boundaries
- Prevents attention from bleeding across unrelated documents

### Sliding Window (-0.0227 additional)
- chunk_size=256 within eval_seq_len=1024
- Each 256-token chunk sees up to 768 tokens of preceding context

### LoRA TTT (-0.0031 additional)
- Per-document LoRA adapters (rank=8) on Q, V projections + lm_head
- Batched implementation: per-batch-element A/B matrices
- **Score-then-train ordering**: chunk is scored BEFORE LoRA trains on it (no data leakage)
- Reset LoRA + optimizer between documents
- Single Adam step per chunk (lr=0.01)
- Uses only ~1/10th of eval budget — room for improvement

### TTT Implementation Details
- `BatchedTTTLoRA` wraps base model
- LoRA applied to: c_q, c_v (every attention block), lm_head
- NOT applied to: keys, MLP, embeddings, norms
- Documents sorted by length for batching efficiency

## Novel Ideas

1. **Batched per-document LoRA**: batch dimension in LoRA matrices enables parallel per-document adaptation. 5x faster than sequential.
2. **Score-then-train causality**: prevents information leakage while allowing within-document adaptation.
3. **Document isolation as separate technique**: +0.011 BPB from simply respecting document boundaries.

## Takeaways

- TTT adds only 0.003 BPB on top of sliding window. Author: "I just wanted to plant the TTT seed."
- Most improvement is from eval engineering (doc isolation + sliding window = 0.034)
- TTT has clear room for optimization (more rank, more layers, more eval budget)
- Architecture is completely unchanged from baseline
